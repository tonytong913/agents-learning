---
name: feedback-learning-style
description: Learning should be dialog-based, not documentation-writing
type: feedback
---

The user wants to learn through discussion, not have notes pre-written for them.
**Why:** The goal is for the user to actually internalize the knowledge, not just have tidy wiki pages.
**How to apply:** Follow SOP: read the plan chapter → discuss in conversation (multi-turn is fine) → only AFTER the chapter is fully discussed, produce/update wiki notes → move to next chapter. Do NOT read all source files silently and dump synthesized notes into the wiki. Act as a study partner/tutor, not a documentation writer.
